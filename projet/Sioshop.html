Projet réalisé en cours : SIO-SHOP



Contexte du projet

SIO-SHOP a été développé en C# Windows Forms, dans le cadre de ma deuxième année de BTS SIO, au cours duquel nous devions mettre en œuvre une application de gestion pour une concession automobile fictive.
L'objectif pédagogique était de consolider les notions de programmation orientée objet (hiérarchie de classes, héritage), de gérer les interactions avec une base de données MySQL.



Technologies utilisées

C# Windows Forms MySQL


L'application offre une authentification sécurisée des utilisateurs, permettant de contrôler l'accès aux différentes fonctionnalités selon le profil de l'utilisateur connecté. Chaque employé dispose d'identifiants personnels pour accéder au système.

Interface d'authentification

Pour la gestion des données, l'application utilise une base de données MySQL qui stocke toutes les informations relatives aux clients, aux produits et aux ventes.
Voici le MCD de cette base de données.

Modèle Conceptuel des Données (MCD) de l'application

Une fois connecté, l'utilisateur accède au menu principal qui propose différentes fonctionnalités : gestion des clients, gestion des produits, saisie des ventes, et déconnexion. L'interface est conçue pour être intuitive et facile à utiliser.

Menu principal avec les différentes fonctionnalités

La fonctionnalité de saisie des ventes permet d'enregistrer les transactions entre les clients et la concession. Elle calcule automatiquement la TVA et met à jour les stocks de véhicules disponibles.

Interface de saisie des ventes

La gestion des clients permet de visualiser, rechercher, ajouter et modifier les informations des clients de la concession. Une barre de recherche facilite la recherche par nom.

Interface de gestion des clients

Un double-clic sur un client permet d'accéder à ses informations détaillées avec la possibilité de les modifier, on voit aussi l'historique de ses achats, ainsi que le montant total dépensé.

Informations personnelles des clients

Le bouton "Nouveau client" nous mène vers cet interface ainsi on peut enregistrer les donées du clients dans la base de données.

Interface d'ajout de client

La gestion des produits permet de visualiser l'ensemble des véhicules disponibles, avec leurs caractéristiques (marque, modèle, prix, stock). Une recherche approfondie par marque est également disponible pour filtrer les résultats.

Interface de gestion des produits

Un double-clic sur un produit permet d'accéder à ses informations détaillées avec la possibilité de les modifier.

Informations des produits

Le bouton "Nouveau Produit" nous mène vers cet interface ainsi on peut enregistrer les donées du produit dans la base de données.

Interface d'ajout de produit

L'application est structurée selon le modèle orienté objet avec une hiérarchie de classes représentant les entités métier : Personne (classe mère), Employé (hérite de Personne), Client (hérite de Personne), et Produit (véhicules). Cette architecture permet une maintenance facile et une évolution du code.
